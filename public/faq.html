JuiceBox FAQ

Everything you usually ask before uploading.

Basics

What is JuiceBox?

A tiny throw-it-in and grab-the-link temporary file host. Drop file → get link → share → it self-destructs later. No accounts needed.

Do I need an account?

Nope. Ownership is tied to the Cloudflare client IP we see. That's only for "can you delete this?", rate limiting and moderation. No profile, no password, no email, just IP.

What is the file size limit?

Up to 500MB per upload. Drop a folder? It's zipped in your browser first, and that ZIP still has to fit the 500MB cap.

Which file types are blocked?

Executables & script-ish stuff (.exe .dll .bat .cmd .com .js .ps1 .sh .php etc.). If it's likely to be weaponized or abused quickly, it's probably blocked.

Can I hotlink files?

Yes. The link you get (/f/<id>) is direct and cacheable until the file expires. Paste freely. When time's up it just 404s.

Expiration

When do files vanish?

You pick a lifespan (1h → 14d). When the timer runs out: delete file + metadata, link goes poof. Like if you never uploaded it.

Can I extend a file's life?

Not yet. Re-upload it with a longer retention if you need more time.

What happens after expiration?

The file is removed, the record is gone, and the link returns 404.

Privacy & Security

Do you log my actual IP?

We store the Cloudflare provided client IP with each upload so you can delete your stuff and so abuse can be curbed plus it's deleted with the uploaded file. No behavioral tracking, selling, or creepy enrichment. If that’s still too much: don’t upload sensitive things. Why are you even concerned for privacy if you use Windows.

Are uploads encrypted?

In transit: yes (TLS/HTTPS). On disk: no. Why? Moderation (need to see what's reported), simplicity, and hotlinked encrypted blobs would be pointless without a key system (which we don't want). So: don't treat it like a vault.

Do you scan malware?

Right now: only basic extension blocking. If abuse spikes, a lightweight hash + VirusTotal style lookup pipeline may be added (will be announced). Basic rules of the internet, Don't run any suspicious things

Can other people list my files?

No public browsing. Someone would need the exact link. Guessing IDs at scale just hits rate limits and sadness.

Abuse & Limits

Rate limits?

Short bursts are fine (token bucket ≈ 60 burst, slowly refills). Might tweak this if abuse emerges

Content rules?

No illegal stuff, no malware, no private personal data you don't have permission to share, no hate / harassment. Edge cases? Ask (or assume "no"). See Terms.

Banned Content

What content is disallowed?

Illegal material, CSAM, sexualized minors, non-consensual intimate imagery, doxxing, malware/exploit droppers, extremist recruitment media, targeted hate, large copyrighted media you don't own, using the service as a long-term CDN dump.

Why these restrictions?

Why are you even asking.

What happens if I upload banned content?

Don't, I'll just delete it and probably IP ban you. Serious stuff may be escalated.

Removal

How do I delete a file early?

Click the ❌ in the UI or send DELETE /d/<filename> from the same network. That's it.

Can staff remove files?

Yes.

API Notes

Is there an API?

POST /upload with multipart file(s) + optional ttl. Returns JSON: { files: ["f/abc123"] }. That's the whole "API."

Retention field values?

1h 3h 12h 1d 3d 7d 14d. Invalid? You get 3d.
